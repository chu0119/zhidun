# 星川智盾 v1.12.0 发布说明

发布日期：2026-05-04

## 版本概览

v1.12.0 聚焦“规则策略工程化”和“分析链路一致性”两大方向，重点提升可配置性、可观测性与运行效率。

## 主要更新

### 1) 规则引擎配置化升级
- 新增规则引擎配置模型，支持：
  - 规则 ID 白名单/黑名单
  - 攻击分类白名单/黑名单
  - 最低风险阈值（info/low/medium/high/critical）
- 规则过滤能力已统一接入：
  - 本地规则分析
  - Worker 分析链路
  - 大文件流式扫描链路
- 分析报告新增规则启用数量信息，便于策略审计。

### 2) 攻击链关联分析增强
- 新增攻击链关联分析段落：按 IP 与行号窗口聚合多步攻击行为。
- 支持攻击链窗口可配置，适配“高敏感排查”与“低误报值守”两类运营场景。

### 3) 性能与复用优化
- 分析流程接入文件快照缓存，重复分析同一文件可快速复用结果。
- 增量与缓存链路打通，减少重复计算，提升大文件分析效率。

### 4) 设置与交互体验增强
- 设置页新增规则策略可视化配置：
  - 风险阈值
  - 攻击链窗口
  - 分析缓存开关
  - 分类/规则 ID 白黑名单
- 新增三种一键策略预设：
  - 高敏感模式
  - 低误报模式
  - 仅高危模式
- 新增“当前策略”状态识别：
  - 设置页显示当前预设/自定义
  - 分析面板同步显示当前策略

### 5) 稳定性与一致性修复
- 完成规则配置链路的 TypeScript 类型收敛。
- 同步修正主进程 IPC 与前端类型声明，避免签名不一致导致的潜在问题。

## 升级建议
- 建议从 v1.11.0 直接升级到 v1.12.0。
- 若你使用的是值守场景，建议优先尝试“仅高危模式”或“低误报模式”。
- 若你处于排查期，建议使用“高敏感模式”并适当扩大攻击链窗口。

## 已知说明
- 本版本不包含破坏性配置变更。
- 旧配置会自动兼容并补齐规则引擎默认参数。
